    • 요즘 주식 하려면 테크 트렌드를 잘 아시는거 같다.
      • 2차전지 얘기…
      • 기술공부를 이걸로 하시는 분들이 많은듯
    • 그런 의미에서 주식 얘기 하신게 아닌가 싶습니다.

  • 석사를 가는게 좋다는 말씀

    • 석사를 가는게 좋다는 얘기 좀 인상적
      • 근데 교수님은 석사 하고 잘풀린 케이스가 그런걸지도…

  • XML

    • 요새 잘 안쓰는 형식인데 신기했다
    • (준배님)예전에 석사때는 모델 구조 짜듯이 사용했다.
      • html5lib 같은것들도 스파크쓸때 아주 많이 사용된다고 들었음.

  • 강화학습

    • 사이버펑크 사례 인상적이었음
    • 강화학습이 딥러닝 들어갔다는 말이 신기했음. 기초적인 소스는 알고있었는데
      • 딥러닝이 어디에 사용되는거지?
      • (준호님)알파고가 강화학습으로 만들었는데, 디테일한 점수체계를 두지 않고 승패에다가 점수를 뒀다.
      • 그래서 안에서 학습하는것들이 딥러닝이었다
    • 하드웨어 쪽으로 발전이 많이 되어있어서 그렇게 생각했음. 하드웨어 제어할때 많이 사용함.
      • 인공지능이 데이터 처리도 있지만, 로봇 등을 움직일때도 사용되니까…
      • 조선소 프로젝트를 보니까 그쪽 분야셔서 발전된게 눈에 보이니까 그런 것일 수도.

  • 생성모델

    • 창(Generation)과 방패(Detective)를 만들어 모델링 하는것
      • 가장 유명한게 GAN(적대적 생성 모델)
      • 논문에서 가장 많이 나왔던 것은 지폐를 위조하려는 도둑과 그걸 잡으려는 경찰
      • 처음에는 랜덤 데이터를 넣었다가, 서로 피드백을 주고 받으면서 엄청 정교해짐
    • 반고흐 화풍으로 그림 만드는 것도 비슷한 방식?
      • 이거는 정확히 transfer learning이고 엄밀히 말하면 GAN이랑 조금 다르다

  • ML엔지니어 관련 이야기

    • 모델 경량화 참 어려운데, 우리같은 초보자들이 할 수 있을까? ML 엔지니어 어렵더라.
    • 데이터 사이언티스트 커리인데 ML 엔지니어를 양성할 수 있을까?
    • 데이터사이언티스트가 되는건 박사 아닌 사람이 되는걸 본적이 거의 없는것같다.
      • 통계학과, 수학과가 진짜 많다
      • (성익님)pabii블로그 공유
      • (지원님) 데이터 사이언티스트는 박사나와도 안되겠구나 하고 생각하고있었는데, 교수님이 말씀하셨듯이 정의가 모호한 것 같다. 사실 개발도 하고 싶고 기회가 되면 연구도 해보고싶다
        • 대학원 갈 생각도 조금은 있고, 석사 안나와도 연구하시는 분들도 있다고 들었습니다.
      • (준배님) 누군가 만들어놓은 논문을 자기 서비스에 구현하는 게 ML 엔지니어라고 하셔서, 관심이 생겼다. 어떤 식으로 사용하고 어떻게 경량화하고 어떻게 서비스라인으로 하는 건지가 궁금해졌다.
      • (현우님) 최성철 교수님 얘기를 듣고, 산업공학과가 좀 맞지 않을까 하는 생각이 들었다. 특히 하드웨어 공부를 하는 파트가… 본인 분야랑 잘 맞는다고 느꼈다. 사실 인공지능 활용(가)에 더 관심이 있다
      • (준호님) 예전에 대학원을 준비했었지만, 알면 알수록 연구보다는 코딩하는게 더 재밌다는게 느껴졌다. 유튜버의 이야기 - 코딩하고 만들어내는거 좋아하는데, 데이터 사이언티스트라는 직업을 가져보니까 위치가 PM-데이터사이언티스트-개발팀의 삼각소통에서 데이터사이언티스트의 역할이 너무 크다. 제품을 만들어내는 성취감도 없고 해서, 오히려 개발을 하고싶다는 생각이 들었다.
